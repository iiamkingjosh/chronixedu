4.1 TypeScript Rules
RULE C1  Strict TypeScript вЂ” no 'any'
RULETypeScript strict mode is enabled. The type 'any' is banned. If a type is 
unknown, use 'unknown' and narrow it. Every function parameter and return 
value must be typed.
  WRONGвќЊfunction getStudent(id: any): any { ... }
  CORRECTвњ…function getStudent(id: string): Promise<Student> { ... }
RULE C2  Import with path aliases only вЂ” no relative paths going up
RULEIn both /apps/web and /apps/api, always use the @/ path alias. Never 
use ../../../ relative imports that go up more than one level.
  WRONGвќЊimport { verifyToken } from '../../../middleware/auth'
  CORRECTвњ…import { verifyToken } from '@/middleware/auth'
RULE C3  Shared types import from @chronixedu/shared
RULETypes used by both frontend and backend must be imported from the shared 
package, never redefined locally.
  WRONGвќЊ// In apps/api/src/routes/students.ts type Student = { id: 
string; name: string; ... }
  CORRECTвњ…import type { Student } from '@chronixedu/shared'
RULE C4  No console.log in production code
RULEUse the Winston logger in the API. Use nothing (silent) in the frontend. 
console.log is for debugging only and must be removed before a prompt is 
considered complete.
  WRONGвќЊconsole.log('Score saved:', score)
  CORRECTвњ…logger.info('Score saved', { scoreId: score.id, userId: 
req.user.userId })
